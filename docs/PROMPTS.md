# AI 协作 Prompt 模板

## 1. 项目总控 Prompt

```text
你是一个资深的 Electron + Node.js + ExcelJS 桌面应用架构师。
当前项目是一个 Excel 自动化工具平台，第一阶段实现 Excel 拆分功能，后续会扩展为合并汇总、去重、对账等能力。
请始终遵循：
1. 先设计配置与模块结构，再生成代码。
2. UI、业务逻辑、Excel 引擎分层清晰。
3. 默认后台执行，不阻塞界面。
4. 所有 sheet 名称、标题行数、拆分列等规则必须配置化。
5. 输出结果保持源表顺序与格式。

当你输出代码时，请优先输出可直接落地的完整模块，而不是伪代码。
```

## 2. 拆分功能 Prompt

```text
请为 Excel 拆分功能生成方案与代码。
需求：
- 用户通过界面选择一个 xlsx 文件
- 读取配置中的多个 sheet
- 每个 sheet 按指定列的值拆分为多个文件
- 每个输出文件保持 sheet 顺序与标题行配置
- 保留尽可能多的格式信息
- 输出目录可配置
- 运行过程需要日志
- 处理过程在后台执行，不阻塞 UI

请先给出数据结构设计、配置结构、模块划分，再输出实现代码。
```

## 3. 合并汇总功能 Prompt

```text
请基于现有 Excel 工具平台设计“多个表合并汇总成一个总表”的功能。
要求：
- 支持多个来源文件或多个 sheet
- 支持字段映射、列合并、去重
- 支持配置化规则
- 输出汇总表保留标题、格式和顺序
- 预留后续对账、透视分析扩展点

请先输出模块设计与配置样例，再输出核心代码。
```

## 4. 配置文件生成 Prompt

```text
请生成一个适用于 Excel 拆分工具的配置文件模板，要求包含：
- 规则启用开关
- sheet 名称
- 标题行数
- 拆分列
- 输出 sheet 名称
- 文件名规则
- 空值处理规则
- sheet 顺序保持规则

输出为 JSON 示例，并解释每个字段用途。
```

## 5. 代码审查 Prompt

```text
请审查以下 Electron / Node.js / ExcelJS 代码，重点检查：
- 是否存在阻塞 UI 的逻辑
- 是否把 UI 与业务逻辑混写
- 是否支持配置化
- 是否保留 sheet 顺序与格式
- 是否存在性能瓶颈
- 是否方便后续扩展

请输出问题清单、风险等级和修改建议。
```

## 6. 生成界面 Prompt

```text
请为 Excel 工具平台生成一个简洁的桌面端首页与拆分页 UI 方案。
要求：
- 首页展示功能入口卡片
- 拆分页支持选择文件、配置规则、选择输出目录、查看日志
- 页面布局清晰、可扩展
- 适合 Electron + Vue 3 + Element Plus

请先给出页面结构说明，再输出可运行的前端代码。
```
