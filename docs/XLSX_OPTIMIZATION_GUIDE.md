# 📊 Excel 文件优化诊断工具

## 三步快速诊断与优化你的 xlsx 文件

### 问题
你的 `test\华锐捷.xlsx` 虽然只有 100+ 行实际数据，但读取很慢，这通常是因为 Excel 文件本身有**隐藏的膨胀问题**。

### 根本原因（按概率排序）

| 原因 | 症状 | 修复难度 |
|------|------|---------|
| **1. 大量空白格式化行列** | Sheet 定义到 1000+ 行但只用前 100 行 | ⭐ 容易 |
| **2. 嵌入图片/图表** | 有隐藏或退缩的图片、OLE对象 | ⭐ 容易 |
| **3. 样式定义冗余** | 数千个未使用的样式 | ⭐⭐ 中等 |
| **4. 合并单元格过多** | 数百个跨行列合并定义 | ⭐⭐ 中等 |
| **5. 共享字符串表肥大** | 重复字符串或超长文本 | ⭐⭐⭐ 困难 |

## 📋 诊断工具详情

### `quick-analyze.py` - 快速扫描
- 查看 ZIP 内部文件大小分布
- 按类别统计 (Sheet、图片、样式等)
- 快速识别占用空间最多的部分
- **耗时: 1-3 秒**

### `deep-diagnose.py` - 深度分析
- 对每个 Sheet 统计行列使用情况
- 检查格式化、合并、图片、样式
- 识别根本原因（空白行、隐藏图片等）
- 计算平均每个单元格的占用空间
- **耗时: 3-10 秒**

### `optimize-xlsx.py` - 自动优化
- 删除超出数据范围的行列
- 移除所有嵌入图片/绘图
- 保留合并单元格和条件格式
- 输出为 `华锐捷_optimized.xlsx`
- **耗时: 10-30 秒**

## 💡 常见问题

### Q: 优化会丢失数据吗？
**A:** 不会。优化只会：
- ✅ 删除**未使用**的空白行列格式化
- ✅ 移除嵌入的**图片/绘图**
- ✅ 保留**所有数据和样式**
- ❌ 不删除任何有值的单元格

### Q: 为什么 Excel 会这么膨胀？
**A:** 常见原因：
1. **Excel 自动扩展**: Excel 打开时常将行数扩展到 1000000
2. **图片嵌入**: 截图、粘贴的图片
3. **格式刷**: 格式被应用到大范围
4. **保存时未压缩**: Excel 不像 LibreOffice 会主动压缩

### Q: 优化后拆分会变快吗？
**A:** **大幅变快！**因为：
- ExcelJS 读取时无需解析 999880 个空白行
- 不需要 clone 这些空白行的格式
- ZIP 解压速度也会快 10-50 倍

### Q: 能用 LibreOffice 打开优化版本吗？
**A:** **完全兼容**。优化版本是标准 xlsx 格式。
