# Excel 自动化桌面工具 PRD

## 1. 项目目标
构建一个基于 **Electron + Node.js + ExcelJS** 的桌面应用，用于处理 Excel 类数据自动化任务。第一阶段重点完成 **按规则拆分 Excel 文件**，后续预留 **多表合并汇总、去重、对账、格式转换、批量处理** 等能力。

项目强调：后台执行、配置驱动、可扩展、可维护。

## 2. 角色与使用场景
- 角色：仓库/供应链/财务/运营人员
- 场景：选择汇总 Excel -> 按配置识别标题行与拆分列 -> 按拆分键输出多个文件 -> 保留顺序与格式 -> 输出到指定目录。

## 3. 第一阶段范围（MVP）
### 必须实现
- 选择源 `.xlsx` 文件
- 配置拆分规则
- 按规则读取多个 sheet
- 以指定列值为拆分键生成多个文件
- 保持原 sheet 顺序与标题行
- 保留常见格式（列宽、行高、字体、边框、填充、对齐、数字格式、合并单元格）
- 指定输出目录
- 运行日志与错误提示

### 暂不实现
- 云端同步、登录系统、数据库存储、自动调度、OCR、在线协作

## 4. 功能需求
### 4.1 文件导入
- 选择本地 `.xlsx`
- 显示文件路径/名称/大小
- 支持重新选择

### 4.2 拆分规则配置
每条规则至少包含：
- `sheetName`
- `enabled`
- `headerRows`
- `splitColumn`
- `splitBy`
- `outputSheetName`

可扩展字段：
- `renameMap`
- `valueTransform`
- `skipEmpty`

### 4.3 拆分逻辑
- 按配置 sheet 顺序处理
- 每个 sheet 内按拆分列值分组
- 同键多行进入同一个输出文件
- 输出文件内 sheet 顺序与源文件一致
- 某键在某 sheet 无数据时仍保留表头

### 4.4 文件输出
- 输出目录可配置
- 文件名默认使用拆分键
- 自动清理 Windows 非法文件名字符
- 冲突策略：覆盖或时间戳

### 4.5 日志与错误
- 记录启动/读取/拆分/导出/完成
- 记录 sheet 缺失、列不存在、权限不足、文件占用等异常
- 提供用户可见运行日志

## 5. 非功能需求
- 性能：大文件处理避免 UI 阻塞
- 稳定性：异常不可导致应用崩溃
- 可扩展：支持后续能力快速接入
- 可维护：模块边界清晰

## 6. 验收标准
- 成功拆分并输出按拆分键命名文件
- 每个文件包含配置 sheet 且顺序正确
- 每个 sheet 标题行保留正确
- 格式尽可能与源表一致
- 日志可查看、错误可定位

## 7. 拆分输出格式兼容矩阵（2026-05）
### 已支持（会迁移）
- 工作表级：视图（含冻结窗格）、页面设置、页眉页脚、自动筛选、分页符
- 行列与单元格：列宽、行高、隐藏、outlineLevel、字体/边框/填充/对齐/数字格式
- 结构对象：合并单元格
- 规则对象：数据验证（Data Validation）、条件格式（Conditional Formatting）

### 未支持（会输出 warning）
- 图表、图片、形状等绘图对象
- 批注/注释（Comments/Notes）
- Table（结构化表对象）

### 说明
- 未支持对象不会阻塞拆分任务；会在日志中标注“当前版本不迁移该对象”，用于帮助用户识别输出差异。
