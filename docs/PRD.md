# Excel 自动化桌面工具 PRD（MVP）

## 1. 项目目标

构建一个基于 **Electron + Node.js + ExcelJS** 的桌面应用，用于处理 Excel 数据自动化任务。  
第一阶段聚焦 **按规则拆分 Excel 文件**，后续预留合并汇总、去重、对账、格式转换、批量处理能力。

核心要求：
- 后台执行，不阻塞 UI
- 配置驱动（sheet、标题行、拆分列等规则可配置）
- 模块化可扩展
- 分层清晰、可维护

## 2. 用户角色与场景

**角色**：仓库 / 供应链 / 财务 / 运营人员

**场景**：
1. 选择包含多个 sheet 的汇总 Excel
2. 按规则识别标题行与拆分列
3. 以拆分字段值生成多个独立 Excel
4. 输出文件保留 sheet 顺序、标题与常见格式
5. 写入指定目录并按规则命名

## 3. MVP 范围

### 3.1 必须实现

- 选择源 `.xlsx` 文件
- 配置拆分规则
- 多 sheet 按规则读取与拆分
- 以指定列值作为拆分键输出多个文件
- 保持原 sheet 顺序
- 保留标题行数
- 尽可能保留格式（列宽、行高、字体、边框、填充、对齐、数字格式、合并单元格）
- 输出到指定目录
- 提供运行日志与错误提示

### 3.2 暂不实现

- 云端同步、登录系统、数据库存储
- 自动调度、OCR、在线协作

## 4. 功能需求

### 4.1 文件导入

- 支持选择本地 `.xlsx`
- 展示文件路径、文件名、大小
- 支持重新选择

### 4.2 拆分规则配置

每条规则至少包含：
- `sheetName`
- `enabled`
- `headerRows`
- `splitColumn`（如 `C`）
- `splitBy`（默认单元格值）
- `outputSheetName`

可扩展字段：
- `renameMap`
- `valueTransform`
- `skipEmpty`

### 4.3 拆分逻辑

- 按配置顺序处理 sheet
- 每个 sheet 按拆分列值分组
- 同一拆分键写入同一输出文件
- 输出文件 sheet 顺序与源文件一致
- 拆分键在某 sheet 无数据时，仍保留该 sheet 表头

### 4.4 文件输出

- 输出目录可配置
- 默认文件名使用拆分键
- 自动清理 Windows 非法文件名字符
- 目标已存在时：覆盖或自动加时间戳

### 4.5 日志与错误

- 记录启动、读取、拆分、导出、完成
- 记录异常：sheet 缺失、列不存在、权限不足、文件占用等
- 提供用户可见日志面板

## 5. 非功能需求

- **性能**：支持大文件处理，避免 UI 阻塞
- **稳定性**：异常不导致应用崩溃
- **可扩展性**：可快速接入后续功能
- **可维护性**：分层清晰，禁止单文件堆叠逻辑

## 6. 验收标准

- 成功按拆分键输出多个文件
- 每个文件包含配置中的 sheet 且顺序正确
- 每个 sheet 标题行数正确
- 拆分结果格式尽可能接近源表
- 运行日志可查看且错误可定位
